Synthèse de présentation

OCR, Regex et Heuristique

Dans ce projet, on lit d'abord le document avec un moteur OCR, puis on extrait les données utiles avec deux approches complémentaires : les regex, qui reconnaissent une forme, et les heuristiques, qui déduisent une information grâce au contexte.

Vue d'ensemble

1. Document
Image ou PDF administratif/commercial.
2. OCR
Le document est converti en texte exploitable.
3. Nettoyage
On corrige les erreurs OCR et les espaces.
4. Extraction
Regex et heuristiques récupèrent les champs métier.
5. JSON final
Les données extraites sont structurées en sortie.

Pourquoi avoir choisi un OCR ?

Définition

OCR = Optical Character Recognition

Un OCR transforme un document visuel, comme une image ou un PDF scanné, en texte exploitable par un programme.

Pourquoi c'est nécessaire
  • un script ne peut pas exploiter directement une image;
  • il faut d'abord récupérer le texte présent dans le document;
  • ce texte sert ensuite de base aux regex et aux heuristiques.
Choix

PaddleOCR

Le projet utilise PaddleOCR comme moteur de reconnaissance de texte.

Avantages
  • lecture d'images et de PDF;
  • bonne qualité sur documents administratifs;
  • intégration simple en Python;
  • utilisation locale, sans service externe.
Pourquoi ce choix
  • open source;
  • adapté à un pipeline de traitement local;
  • fournit un texte directement réutilisable dans le projet.
Idée simple à dire : PaddleOCR est la première étape du pipeline. Sans OCR, il n'y a pas de texte à analyser.

Les regex : à quoi ça sert ?

Définition

Une regex, ou expression régulière, est un motif qui décrit la forme attendue d'une donnée.

  • \d{14} pour un SIRET;
  • [0-9]{2}/[0-9]{2}/[0-9]{4} pour une date;
  • [A-Z0-9]{8,11} pour un BIC.
Pourquoi on les utilise
  • elles sont précises;
  • elles sont rapides;
  • elles marchent très bien quand le format est stable.

Règles de base

  • \d = un chiffre;
  • \s = un espace ou retour ligne;
  • + = un ou plusieurs;
  • * = zéro ou plusieurs;
  • ? = optionnel;
  • {9} = exactement 9 fois;
  • (...) = groupe capturé.

Exemple simple

Num[ée]ro de facture\s*:?\s*([A-Z0-9-]+)
  • Num[ée]ro accepte plusieurs écritures;
  • \s* accepte les espaces;
  • :? rend le : optionnel;
  • le groupe final capture le numéro de facture.

Quand les utiliser

  • sur les documents très structurés;
  • quand les libellés sont fixes;
  • quand le format de la donnée est connu d'avance.

Exemples : URSSAF, KBIS, Attestation SIRET.

Les heuristiques : comment ça fonctionne ?

Définition

Une heuristique est une règle pratique basée sur le contexte, la position ou l'ordre des éléments dans le document.

Pourquoi on en a besoin
  • certains documents changent beaucoup selon l'émetteur;
  • l'OCR coupe parfois le texte en plusieurs morceaux;
  • une regex unique serait trop fragile dans ces cas-là.

Exemple concret

Sevenit GmbH
Hauptstraße 40
77654 Offenburg
Monsieur Jean Dupont
Acheteur SA
Rue du Chateau
34000 Montpellier
FACTURE
Description
...
Total HT

Règle heuristique

  • le mot FACTURE sert de repère;
  • les blocs avant FACTURE contiennent vendeur et client;
  • le premier bloc correspond au vendeur;
  • le second bloc correspond au client.

Autres exemples

  • entre Description et Total HT = lignes de facture;
  • entre Domiciliation et IBAN = zone bancaire;
  • un token comme 34000 Montpellier ressemble à une ville.
Les heuristiques sont surtout utiles sur les fichiers les moins standardisés : RIB, Facture et Devis.

La différence essentielle

Regex Je reconnais une forme.
Heuristique Je déduis une information grâce à la position et au contexte.

Quand utiliser quoi ?

Regex
  • données au format stable;
  • documents officiels;
  • libellés bien identifiables.
Heuristique
  • documents semi-structurés;
  • mise en page variable;
  • texte OCR moins propre.
En pratique

Dans le projet, les deux approches sont complémentaires : l'une apporte la précision, l'autre apporte la souplesse.
